What's under the hood

The model, by the numbers

Everything here is measured straight from the trained model by evaluate.py, not typed in. Test accuracy comes from a held-out split the model never saw while training or picking its checkpoint, so it shows how it does on new fruit, not what it memorized. The test set is small (44 images), so the 95% confidence interval is wide.

How it ships

From photos to a live model

The whole path: train in PyTorch, check it honestly, export to ONNX, and run it in the browser with WebAssembly.

Tech stack
Deployment
static page (Vercel CDN)
   |  canvas -> Float32 tensor [1,3,64,64]
   v
onnxruntime-web (WebAssembly)
   |  TinyFreshNet.onnx, one forward pass
   v
softmax -> { class, confidence, timing }

No backend. The model is trained in PyTorch, exported to ONNX, and runs fully in the browser, so the demo stays up with no cold starts. A FastAPI server is in the repo if you would rather run it on a machine.

Honest, not impressive-sounding.

The headline is held-out test accuracy with its confidence interval, not the training score. Validation reads high because it picks the checkpoint, so it is kept off to the side rather than sold as the result.
